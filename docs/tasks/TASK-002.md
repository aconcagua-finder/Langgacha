# TASK-002: Бустеры + Коллекция

## Цель

Реализовать открытие бустера (5 карточек за раз) с анимацией и страницу коллекции с фильтрами. Пока без авторизации — все карты принадлежат одному условному игроку.

Результат: игрок нажимает "Открыть бустер" → видит анимацию раскрытия 5 карт → карты попадают в коллекцию → на отдельной странице видна вся коллекция с фильтрами.

---

## 1. Backend: API бустера

### POST /api/boosters/open

Генерирует 5 карточек за один запрос.

**Логика выбора рарности каждой карты:**

Используй таблицу шансов из GDD (раздел 5):
- C: 50%
- UC: 30%
- R: 13%
- SR: 5%
- SSR: 2%

**Гарантии:**
- Минимум 1 карта из 5 должна быть UC или выше. Если при случайном броске все 5 выпали C — перебрось последнюю карту с шансами только UC+.

**Алгоритм:**
1. Для каждой из 5 карт — бросить рарность по таблице шансов
2. Проверить гарантию (хотя бы 1 не-C)
3. Для каждой рарности — выбрать случайное слово этой рарности из таблицы Word
4. Рандомизировать статы (уже реализовано в cards.service — rollStat)
5. Создать 5 записей Card в БД
6. Вернуть массив из 5 карточек

**Ответ:** массив из 5 объектов GeneratedCardDto (тот же формат что у POST /api/cards/generate).

### Структура файлов

Создать новый модуль:
```
backend/src/modules/boosters/
├── boosters.routes.ts
├── boosters.service.ts
└── boosters.types.ts
```

Вынести общую логику создания карточки (выбор слова, rollStat, создание Card) из `cards.service.ts` в переиспользуемую функцию, чтобы и `/api/cards/generate`, и `/api/boosters/open` использовали один и тот же код генерации.

Шансы рарности вынести в `shared/constants.ts`.

## 2. Backend: улучшение GET /api/cards

Добавить query-параметры фильтрации:
- `type` — фильтр по типу карты (Persona, Lugar, Acción, Objeto, Emoción, Expresión)
- `rarity` — фильтр по рарности (C, UC, R, SR, SSR)
- `sort` — сортировка: `newest` (по дате, по умолчанию), `fue_desc`, `def_desc`, `rarity_desc`

Пример: `GET /api/cards?type=Acción&rarity=R&sort=fue_desc`

## 4. Frontend: страница бустера (HomePage → BoosterPage)

Переделать текущую HomePage в страницу бустера. Кнопка "Открыть бустер" вместо "Сгенерировать карточку".

### Анимация открытия бустера

Поэтапное раскрытие 5 карточек:

1. **Состояние "пак":** отображается стилизованная обложка пака (бустера). Кнопка "Открыть".
2. **Состояние "раскрытие":** после нажатия — карточки появляются по одной с интервалом ~0.5с. Каждая карта появляется с эффектом (scale от 0 к 1 + лёгкий glow по рарности). Карточки выстраиваются в горизонтальный ряд (или веер).
3. **Состояние "просмотр":** все 5 карт видны. Каждая кликабельна — переворот на обратную сторону. Под карточками — кнопка "Открыть ещё" (пока без ограничений на количество).

Не нужно делать сложную анимацию разрыва пака — достаточно красивого последовательного появления карточек. Фокус на ощущении "что выпало" — подсветка рарных карт ярче.

### Визуал пака

Простой прямоугольник с градиентом, логотипом "LangGacha" и количеством карт (×5). Не нужно сложной графики.

## 5. Frontend: страница коллекции (CollectionPage)

### Сетка карточек

- Адаптивная сетка: 2-4 карточки в ряд в зависимости от ширины экрана
- Карточки отображаются уменьшенными (масштаб ~70% от полного размера)
- По клику на карточку — модалка с полноразмерной карточкой (с переворотом)

### Фильтры

Панель фильтров сверху:
- По типу: кнопки-фильтры с иконками типов (👤📍⚡🔧💫🗣️). Можно выбрать несколько.
- По рарности: кнопки C / UC / R / SR / SSR с цветами рарности. Можно выбрать несколько.
- Сортировка: выпадающий список (Новые, По силе, По защите, По рарности)

### Счётчики

Над сеткой показать:
- Всего карт: N
- Уникальных слов: (пока можно просто считать по word)

### Пустое состояние

Если коллекция пуста — сообщение "Коллекция пуста. Открой первый бустер!" со ссылкой на страницу бустера.
