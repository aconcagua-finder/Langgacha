# TASK-003: Боевая система (5v5 PvE)

## Цель

Реализовать пошаговый 5v5 бой с ботом: выбор колоды → квиз перед каждой парой → автоматический расчёт боя → результаты и награды.

Результат: игрок выбирает 5 карт из коллекции, начинает бой, отвечает на вопросы, смотрит как карты сражаются, получает награды.

### Типы (battle.types.ts)

```typescript
type BattleCard = {
  id: string;           // ID карточки (Card)
  word: string;
  translationRu: string;
  type: string;
  rarity: string;
  fue: number;          // итоговая FUE (после бафов состояния)
  def: number;          // итоговая DEF (после бафов состояния)
  hp: number;           // DEF × 1.5
  condition: string;
  quizCorrect: string;
  quizOptions: string[];
};

type RoundResult = {
  roundNumber: number;           // 1-based
  playerCard: BattleCard;
  botCard: BattleCard;
  quizCorrect: boolean;          // ответил ли игрок правильно
  inspiracionApplied: boolean;   // +15% FUE
  combatLog: CombatTick[];       // пошаговый лог ударов
  winner: "player" | "bot";
  survivorHpLeft: number;        // сколько HP осталось у выжившей карты
};

type CombatTick = {
  attacker: "player" | "bot";
  damage: number;
  targetHpAfter: number;
};

type BattleResult = {
  id: string;
  rounds: RoundResult[];
  winner: "player" | "bot";
  rewards: BattleRewards;
};

type BattleRewards = {
  polvo: number;
  bonusCard: GeneratedCardDto | null;  // 20% шанс
  correctAnswers: number;
  streak: number;                       // максимальная серия правильных ответов
};
```

### POST /api/battle/start

Тело запроса:
```json
{
  "cardIds": ["uuid1", "uuid2", "uuid3", "uuid4", "uuid5"]
}
```

Валидация:
- Ровно 5 карт
- Все карты существуют в БД
- Нет дубликатов

Ответ: объект с подготовленными данными для боя (ID боя, карты игрока, карты бота, но **без ответов квиза** — чтобы клиент не мог подсмотреть).

```json
{
  "battleId": "uuid",
  "playerCards": [ { ... без quizCorrect } ],
  "botCards": [ { ... } ],
  "rounds": [
    {
      "roundNumber": 1,
      "playerCard": { "id": "...", "word": "casa", ... },
      "botCard": { "id": "...", "word": "...", ... },
      "quiz": {
        "question": "Как переводится «casa»?",
        "options": ["дом", "машина", "улица", "город"]
      }
    }
  ]
}
```

**Важно:** правильный ответ квиза НЕ отправляется клиенту. Клиент отправляет ответ на сервер, сервер проверяет.

### POST /api/battle/:battleId/answer

Тело запроса:
```json
{
  "roundNumber": 1,
  "answer": "дом"
}
```

Ответ: результат раунда (RoundResult) — правильно ли ответил, лог боя, кто победил в раунде.

Если это последний раунд (или одна из сторон полностью проиграла) — в ответ также включается финальный BattleResult с наградами.

### Хранение состояния боя

Бой — короткоживущая сущность. Хранить в **Redis** (TTL 30 минут). Ключ: `battle:{battleId}`.

Структура в Redis:
```json
{
  "playerCards": [...],
  "botCards": [...],
  "currentRound": 1,
  "rounds": [],
  "playerSurvivor": null,
  "botSurvivor": null,
  "correctStreak": 0,
  "totalCorrect": 0
}
```

Не нужна отдельная таблица в PostgreSQL для боёв (на MVP). Если Redis недоступен — fallback на in-memory Map (с предупреждением в логе).

## 2. Генерация колоды бота (battle.bot.ts)

Бот получает колоду, подобранную по силе колоды игрока:

1. Посчитать суммарную силу карт игрока: `totalPlayerPower = sum(fue + def)` для всех 5 карт
2. Целевая сила бота: `totalPlayerPower * (0.8 + Math.random() * 0.4)` — то есть ±20%
3. Из seed-пула (таблица Word) выбрать 5 случайных слов
4. Для каждого слова сгенерировать карту с рандомизацией статов (через `generateCardFromPool`)
5. Карты бота НЕ сохраняются в таблицу Card — это виртуальные карты только для боя (генерировать данные in-memory или сохранять временно)

**Упрощение:** на MVP бот не подстраивает каждую карту индивидуально — просто генерирует 5 случайных и масштабирует их статы к целевой силе. Это достаточно хорошо для начала.

## 3. Боевая механика (battle.combat.ts)

### Модификаторы состояния карточки

Перед боем к статам карты применяется модификатор состояния:

| Состояние | Модификатор |
|-----------|-------------|
| Brillante | ×1.10 |
| Normal | ×1.00 |
| Gastada | ×0.90 |
| Deteriorada | ×0.75 |

### Баф Inspiración

Если игрок правильно ответил на квиз:
- FUE карты × 1.15 (на этот раунд)

### Расчёт HP

`HP = effectiveDEF × 1.5` (для обеих сторон)

### Расчёт урона

Формула удара:
```
damage = max(1, attackerFUE - defenderDEF * 0.5)
```

- Минимальный урон всегда 1 (бой не может зависнуть)
- Карты бьют по очереди: сначала игрок, потом бот, потом игрок и т.д.
- Бой продолжается пока одна карта не упадёт (HP <= 0)

### Выжившая карта

Карта-победитель раунда переходит в следующий раунд **с оставшимся HP**. Если у игрока карта выжила с 50 HP — она встречает следующую карту бота, начиная с 50 HP.

### Пример раунда

```
Игрок: casa (FUE=248, DEF=335, HP=502, condition=Normal)
Квиз: "дом" — правильно! → FUE=248*1.15=285
Бот: libro (FUE=300, DEF=280, HP=420)

Тик 1: Игрок бьёт → damage = max(1, 285 - 280*0.5) = 145 → бот HP: 420-145=275
Тик 2: Бот бьёт → damage = max(1, 300 - 335*0.5) = 133 → игрок HP: 502-133=369
Тик 3: Игрок бьёт → 145 → бот HP: 275-145=130
Тик 4: Бот бьёт → 133 → игрок HP: 369-133=236
Тик 5: Игрок бьёт → 145 → бот HP: 130-145=-15 → бот пал!

Победитель раунда: игрок (HP осталось: 236)
```

## 4. Награды (battle.rewards.ts)

### За бой в целом (при победе)

Polvo = сумма по побеждённым картам бота:
- C: 2 Polvo
- UC: 5 Polvo
- R: 10 Polvo
- SR: 40 Polvo
- SSR: 200 Polvo

### За правильные ответы (независимо от победы)

- Каждый правильный ответ: +5 Polvo
- Серия 3+ правильных подряд: множитель ×1.5 на бонус за ответы

### Бонусная карта

При победе — 20% шанс получить случайную карту (через `generateCardFromPool`). Эта карта сохраняется в БД.

### Мастерство

При правильном ответе — `masteryProgress += 1` для соответствующей карты игрока (обновить в БД). Максимум 5.

**DeckSelect:**
- Показать все карты из коллекции (маленькие)
- Игрок кликает на карту — она добавляется в слот колоды (5 слотов внизу)
- Клик на карту в слоте — убирает обратно
- Кнопка "Начать бой" активна когда выбрано ровно 5 карт
- Порядок выбора = порядок в бою (первая выбранная идёт первой)

### Фаза 2: Бой (BattleArena)

Последовательность для каждого раунда:

1. **Показать пару** — карта игрока слева, карта бота справа. Между ними — VS
2. **Квиз (QuizPhase)** — внизу появляется вопрос с 4 вариантами. Игрок выбирает ответ → отправка на сервер
3. **Результат квиза** — зелёная/красная подсветка ответа. Если правильно — на карте игрока появляется маркер Inspiración (+15%)
4. **Бой (CombatPhase)** — пошаговая анимация ударов. Показать HP-бары у обеих карт. Каждый тик: полоска HP уменьшается, число урона всплывает над картой. Интервал между тиками ~0.8с
5. **Результат раунда (RoundResult)** — кто победил, сколько HP осталось. Кнопка "Следующий раунд" (или автопереход через 2с)

### Фаза 3: Результаты (BattleResult)

- Победа/поражение (крупный текст)
- Статистика: правильных ответов X/5, максимальная серия
- Награды: сколько Polvo, бонусная карта (если выпала — показать с анимацией)
- Кнопки: "Ещё бой" (→ выбор колоды), "В коллекцию"

### Визуал HP-бара

Простой горизонтальный бар:
- Зелёный при HP > 50%
- Жёлтый при HP 20-50%
- Красный при HP < 20%
- Числовое значение HP над баром

### Визуал урона

При ударе — число "-145" всплывает над картой и исчезает (CSS анимация, ~1с).

## Критерии готовности

1. `POST /api/battle/start` принимает 5 cardIds, генерирует колоду бота, возвращает раунды с квизами (без ответов)
2. `POST /api/battle/:id/answer` принимает ответ, проверяет, возвращает лог боя раунда
3. Карта-победитель переходит в следующий раунд с оставшимся HP
4. После последнего раунда — финальный результат с наградами (Polvo, бонусная карта)
5. Мастерство карт обновляется в БД при правильных ответах
6. Состояние боя хранится в Redis (TTL 30 мин)
7. На фронте: выбор 5 карт из коллекции → начало боя
8. Квиз с 4 вариантами ответа, визуальная обратная связь (правильно/неправильно)
9. Анимация пошагового боя: HP-бары, числа урона
10. Экран результатов с наградами
11. `docker compose up` — всё работает
