## 2. DeckSelect с фильтрами и sticky deck bar

### Текущее состояние

`DeckSelect` (в `frontend/src/components/battle/DeckSelect.tsx`) — плоский список всех карт без фильтров. Выбранная колода (5 слотов) и кнопка "Начать бой" скроллятся вместе с контентом.

### Что нужно

Переработать DeckSelect:

1. **Добавить фильтры** — переиспользовать `CollectionFilters` (уже есть, работает). Добавить state для `selectedTypes`, `selectedRarities`, `sort` и передавать их в API `listCards(...)`. Точно так же как в `CollectionPage`.

2. **Sticky deck bar снизу** — выбранные карты + кнопка "Начать бой" закрепить внизу экрана:

```
┌─────────────────────────────────────────┐
│ [фильтры]                               │  ← скроллится
│ [сетка карточек]                         │  ← скроллится
│                                          │
├─────────────────────────────────────────┤
│ [слот1] [слот2] [слот3] [слот4] [слот5] │  ← fixed bottom
│            [Начать бой]                  │
└─────────────────────────────────────────┘
```

### Реализация sticky deck bar

Вынести нижнюю часть (5 слотов + кнопка) в `fixed bottom-0` контейнер, аналогично тому, как была сделана кнопка боя в коллекции (но с контентом слотов).

Структура sticky bar:
- Контейнер: `fixed bottom-0 left-0 right-0 z-20 border-t border-slate-800/60 bg-slate-950/80 backdrop-blur`
- Внутри: горизонтальный ряд из 5 мини-слотов (маленькие превью ~60×90 или подобного размера — НЕ полный CardMini) + кнопка
- Пустой слот: рамка с номером
- Заполненный слот: миниатюра карты (слово + рарность-цвет рамки), клик = убрать из колоды
- Кнопка "Начать бой" справа от слотов, disabled пока не 5 карт

Не используй компонент CardMini для слотов в sticky bar — он слишком большой (200×320). Сделай простые маленькие превью прямо в DeckSelect (или вынеси в отдельный компонент `DeckSlot.tsx` если получается больше 30 строк).

**Важно:** добавь `pb-[...px]` к основному контенту DeckSelect, чтобы карточки не прятались за sticky bar. Высоту паддинга подбери под высоту sticky bar.

### Карточки в сетке

Для отображения карточек в сетке DeckSelect используй `CardMini` как сейчас. Выбранные карты подсвечиваются рамкой `border-sky-400/60` — это уже работает, оставить как есть.

### Группировка дубликатов

Сейчас DeckSelect показывает каждую карту отдельно (без группировки). **Оставить так** — при выборе колоды игрок выбирает конкретные экземпляры карт, а не слова. Группировка здесь не нужна.

## 3. Бот — подбор по рарности

### Текущее состояние

`backend/src/modules/battle/battle.bot.ts` — бот берёт 5 случайных слов из пула и масштабирует общую силу ±20%. Но рарность карт случайная — если игрок выбрал 5 SSR, бот может получить 5 C с накрученными статами. Это выглядит нелогично.

### Что нужно

Бот должен подбирать рарности, **похожие** на колоду игрока. Не зеркальные (не копировать 1:1), а примерно того же уровня.

### Алгоритм

1. Посчитать "средний ранг рарности" колоды игрока: C=0, UC=1, R=2, SR=3, SSR=4. Среднее по 5 картам.
2. Для каждой из 5 карт бота: сгенерировать рарность ±1 ранг от среднего игрока (случайно).
3. Выбирать слова из пула, у которых рарность совпадает с целевой. Если нет слов нужной рарности — брать ближайшую доступную.
4. Масштабирование силы ±20% оставить как есть.

Пример: игрок средний ранг 2.4 (несколько R и UC). Бот получает карты рарности 1-3 (UC, R, SR) в разных комбинациях.

### Где менять

`backend/src/modules/battle/battle.bot.ts` — функция `generateBotDeck`.
