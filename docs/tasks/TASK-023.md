# TASK-023: Расширение seed-пула до 150 слов + conceptKey для мульти-языка

**Важно:** все предыдущие задачи (TASK-001–022) уже выполнены и проверены. Не нужно их восстанавливать. Путь к этому файлу: `docs/tasks/TASK-023.md`.

## Контекст

Сейчас в пуле 30 испанских слов. Этого мало для тестирования баланса и реального обучения. Нужно расширить до ~150 слов с осмысленным распределением по типам, рарности и тематике.

Кроме того, нужно заложить фундамент для мульти-языковой поддержки: добавить поле `conceptKey` в модель Word. Это строковый ключ (например "house", "friend", "run"), который привяжет слова на разных языках к одному концепту. В будущем, когда добавим английский, картинки будут привязаны к conceptKey — одна картинка для "casa" (es) и "house" (en).

## 1. Миграция: добавить conceptKey в Word

### Prisma-схема

В модели `Word` добавить:
```prisma
conceptKey    String?
```

Поле опциональное (nullable), чтобы миграция прошла без проблем на существующих данных. Позже сделаем required когда все слова будут с ключами.

### Миграция
```bash
npx prisma migrate dev --name add-concept-key
```

В `seed-words.ts` — добавить `conceptKey` в тип `SeedWord` и в данные каждого слова. Для существующих 30 слов — проставить conceptKey.

В `seed.ts` — добавить `conceptKey` в upsert create/update.

## 2. Расширение seed-пула до ~150 слов

### Распределение по рарности
- **C (Common):** ~75 слов — базовая лексика (числа, цвета, дни недели, еда, семья, части тела, одежда)
- **UC (Uncommon):** ~45 слов — полезная лексика (профессии, транспорт, природа, бытовые действия)
- **R (Rare):** ~20 слов — продвинутая лексика (абстрактные понятия, идиомы)
- **SR (Super Rare):** ~10 слов — сложная лексика (фразовые глаголы, культурные выражения)

Это примерно соответствует шансам в бустерах (50/30/13/5), чтобы был адекватный пул для каждой рарности.

### Распределение по типам
Примерно равномерно по 6 типам (Person, Place, Action, Object, Emotion, Expression), с естественным перекосом (Action и Object побольше — там больше базовой лексики).

### Требования к каждому слову

Каждое слово должно содержать:
- `conceptKey` — англоязычный ключ концепта (lowercase, snake_case для составных: `thank_you`, `good_morning`)
- `word` — испанское слово
- `translationRu` — русский перевод
- `type` — один из 6 типов
- `rarity` — C/UC/R/SR
- `baseAtk`, `baseDef` — базовые статы (процент в диапазоне рарности, от 20 до 80)
- `colorido` — яркость 1-5
- `flavorText` — пример использования на испанском (естественное предложение, разговорный стиль)
- `hint` — подсказка на русском (краткая, полезная для запоминания)
- `tags` — тематические теги на испанском (2-3 штуки)
- `quizCorrect` — правильный ответ на русском
- `quizOptions` — 4 варианта (правильный + 3 дистрактора одного типа/темы)

### Качество контента

**flavorText** — это ключевой элемент обучения! Примеры должны быть:
- Естественными (разговорный испанский, не учебниковый)
- Контекстными (показывают как слово используется в реальности)
- Запоминающимися (яркие ситуации лучше скучных)

**quizOptions** — дистракторы должны быть правдоподобными:
- Того же типа (не мешать «стол» с «бежать»)
- Похожей тематики (для «красный» дистракторы — другие цвета, не «стул»)
- НЕ случайные слова

**hint** — краткий, полезный:
- Мнемоника, этимология или грамматический факт
- Не повторять то, что уже видно на карте

### Формат данных

Данные идут в `backend/scripts/seed-words.ts`. Файл будет большим (~1500+ строк) — это нормально, это данные, не логика. Можно разбить на несколько файлов по рарности если удобнее:
- `seed-words-common.ts`
- `seed-words-uncommon.ts`
- `seed-words-rare.ts`
- `seed-words-sr.ts`

И в `seed-words.ts` — реэкспорт объединённого массива.
