## 1. Пересоздать сегодняшнего рейд-босса

Текущий босс (HP 13775, DEF 2321) создан до фикса. С новой адаптивной генерацией он должен быть значительно слабее.

### Что сделать

Удалить сегодняшнюю запись RaidDay из БД и все связанные RaidAttack, чтобы при следующем запросе рейда он пересоздался с новыми адаптивными статами.

**Способ:** добавить эндпоинт в dev-маршруты или использовать Prisma Studio / SQL-запрос:

```sql
DELETE FROM "RaidAttack" WHERE "raidDayId" IN (SELECT id FROM "RaidDay" WHERE date = CURRENT_DATE::text);
DELETE FROM "RaidDay" WHERE date = CURRENT_DATE::text;
```

Или если есть dev-маршруты — добавить `DELETE /api/dev/raid-reset` который удаляет сегодняшний рейд.

### Важно
- После удаления, первый GET `/api/raid` создаст нового босса с адаптивными статами
- Это разовая операция, но полезно иметь dev-эндпоинт для тестирования

## 2. Мини-карты в бустере — убрать compact-режим

### Проблема
В `frontend/src/components/booster/BoosterRevealGrid.tsx` строка 116:
```tsx
<CardMini card={card} size="booster" />
```

`size="booster"` включает compact-режим: скрывает тип (текст), перевод, состояние, мастерство. Карты выглядят урезанно по сравнению с коллекцией.

### Решение

Заменить `size="booster"` на `size="mini"` в BoosterRevealGrid.tsx строка 116:
```tsx
<CardMini card={card} size="mini" />
```

**Также** нужно увеличить контейнер карты (строка 115), потому что size="mini" имеет aspect-ratio `200/320` и нужно больше места:

```tsx
<div className="w-[120px] min-[375px]:w-[130px] sm:w-[150px] md:w-[170px] lg:w-[190px] xl:w-[200px]">
```

Убрать фиксированную высоту — CardMini с size="mini" сам задаёт aspect-ratio.

### Проверка
- В бустере карты должны выглядеть идентично мини-коллекции: тип с текстом, перевод, ATK/DEF, состояние, мастерство
- Сетка 3 колонки на мобиле, 5 на десктопе — сохранить
- Карточки не должны обрезаться
